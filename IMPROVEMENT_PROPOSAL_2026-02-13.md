# DeepValue-AI-Agent 改进提案（2026-02-13）

## 1. 背景与目标
当前系统已完成 Step1~Step4 的可运行闭环，并具备 AkShare → Lixinger → Baostock 的数据源切换能力。
本提案目标是：在保持“防御型深度价值”风格前提下，提升稳定性、解释性、以及实盘可执行性。

## 2. 当前状态（今日基线）
- Step1 候选数：12（由此前 1 显著提升）
- Step2 通过数：10（由此前 0 显著提升）
- 报告链路：已可自动生成 Markdown 报告与 LLM Prompt
- 数据源状态：AkShare 当前网络不可用；Lixinger 当前路径仍待联调；Baostock 可稳定运行

## 3. 关键不足（按影响排序）
1) 数据源可用性与一致性不足：Lixinger 尚未完全打通，跨源字段口径尚无统一校准。
2) 筛选策略虽已放宽，但参数治理分散：门槛和行业阈值仍在代码中，缺少统一配置中心。
3) 风险排雷能力偏弱：舆情工具仍为占位实现，A/H 溢价模型为简化口径。
4) 现金流质量评估维度单一：仍以 OCF/净利润为主，缺少多期稳定性和行业分位对比。
5) 工程可观测性不足：缺少每轮筛选日志、规则淘汰率统计、数据源健康看板。

## 4. 改进路线图

### P0（1周内，必须完成）
- 配置中心化：新增 strategy_config.json，承载所有阈值与行业规则。
- 数据源诊断模式：为 Lixinger 增加路径探测、响应 code/msg 记录、失败原因分类。
- 结果可解释性：输出“入选/淘汰原因列”，并生成筛选统计摘要（每条规则淘汰比例）。
- 缓存快照：按日期落地原始数据与筛选中间层，支持离线复跑。

### P1（2~3周，优先推进）
- 舆情工具实装：接入公告/新闻检索源，替换当前 placeholder。
- 财务质量增强：引入 3 年滚动 OCF 稳定性、净利润波动、分红连续性指标。
- 行业阈值优化：从“人工关键词阈值”升级为“行业分位数动态阈值”。

### P2（3~6周，中期建设）
- 回测闭环：增加组合层回测（交易成本、滑点、最小手数约束）。
- 风控引擎：加入分红下修、估值反转、现金流恶化的自动降仓规则。
- 监控看板：每日输出数据源成功率、筛选产出数、风险触发数。

## 5. 验收指标（量化）
- 数据层：主链路日可用率 ≥ 95%，切换后可用率 ≥ 99%。
- 筛选层：单次候选数量稳定在 8~30（避免过稀或过宽）。
- 风险层：Step2 通过率稳定在 40%~80% 区间（可解释且可控）。
- 工程层：每轮产出可追溯日志（数据源、阈值、淘汰原因）完整率 100%。

## 6. 下一次开发会话建议起点
1) 先落地 strategy_config.json 与配置加载器。
2) 再做 Lixinger 诊断模式并修复路径/参数差异。
3) 最后补筛选统计报表（rules_summary.csv + run_summary.md）。

## 7. 今日结论
系统已从“可运行”进入“可优化”阶段。
短期重点应放在“数据源稳定 + 配置治理 + 风险工具实装”三件事，先把可用性和可解释性做扎实，再扩展策略复杂度。

## 8. 参考开源仓库（建议纳入持续借鉴清单）

1) qiansen1386/lixinger-openapi
- 用途：理杏仁 Open API 的 Python 封装参考。
- 当前借鉴点：query_json/query_dataframe 的调用方式、路径写法（点号转斜杠）、token 管理。
- 本项目落地：Lixinger 适配器已改为“优先 SDK 语义 + HTTP 回退 + 分批请求聚合”。

2) microsoft/qlib
- 用途：量化研究工作流与数据层标准化。
- 可借鉴点：数据集组织、实验配置、可复现实验追踪。

3) mementum/backtrader
- 用途：策略回测与组合执行框架。
- 可借鉴点：交易成本建模、仓位控制、风控规则可回测验证。

4) ricequant/rqalpha
- 用途：A股语境下的事件驱动回测框架。
- 可借鉴点：A股交易规则处理、调仓与回测评估。

5) vnpy/vnpy
- 用途：事件驱动交易与风控工程实践。
- 可借鉴点：模块化网关、监控告警、任务调度与日志体系。

6) AI4Finance-Foundation/FinGPT
- 用途：金融文本理解与情绪事件抽取。
- 可借鉴点：将新闻/公告文本转为结构化风险信号，增强 Step3 排雷质量。

## 9. 针对本项目的借鉴优先级
- 第一优先级：qiansen1386/lixinger-openapi（直接提升数据源可用性）。
- 第二优先级：qlib + backtrader（把“筛选”升级为“可验证策略”）。
- 第三优先级：FinGPT（把占位舆情工具升级为可用风险信号）。
